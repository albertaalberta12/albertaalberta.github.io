STUDENT ATTENDANCE MONITORING SYSTEM

Development of Automated Student Attendance Monitoring System using Image Processing, is a prototype that automatically records attendance upon registration of the student to the system. It aims to minimize the time execution and to outsmart the traditional attendance system like roll-calls, filling signatures, and timeclocks. The system lessens the time execution in acquiring attendance record because it records studentsâ€™ information and their faces patterns, which serves as verification data to generate an attendance report.

The system was developed using Python programming language for the construction of codes, PyQt5 designer for the graphical user interface and Numpy and OpenCV for libraries, with Local Binary Pattern Histogram algorithm for the face recognition prediction and SQLite for the database management.

Purpose

The concept of the Automated Student Attendance Monitoring System using Image Processing was inspired by the school attendance environment. Attendance is considered a major requirement for students not only for their learning as they have their presence in the class, but also a troublesome for instructors specifically on record-keeping. We identified the existing attendance system that they have been used such as name-calling, submission of index cards, and signatories. We concluded that it is very time-consuming for them that affects the time duration of the class regardless of different subjects. This study implements a solution to numerous issues when it refers to attendance, but before we did the study, first we identified what will be an effective approach of methodology will be used in order to have a guide and key pointers that will aid you through the research. We chose Agile Development Method as we researched the different approaches this method was considered by ourselves to the study because agile development is an efficient strategy which consists of 6 stages that form a cycle, it means the end product can be iterated depending on situation, necessity and sudden changes. On the first stage, we gather possible requirements: the background of the study, to identify what will be the criteria, the objectives, the significance, and the scope and limitations.
